# Client Management

The Clients page is your central hub for managing all your client relationships, tracking their connection status, and maintaining their integration settings.

## Overview

<Frame>
  ![Clients Overview](/images/clients-overview.png)
</Frame>

The Clients page provides a comprehensive view of all your clients, their connection status, and essential integration details, helping you maintain strong client relationships and ensure smooth content delivery.

## Quick Actions

Located at the top of your Clients page, you'll find the essential action:

- **Add New Client**: Quickly onboard new clients with their WordPress and Koala integration details

<Tip>
  Use the "Add New Client" button to streamline your client onboarding process - all essential integrations are set up automatically.
</Tip>

## Client Table Overview

<Frame>
  ![Client Table](/images/client-table.png)
</Frame>

The Client Table displays key information about each client:

- Client name and unique identifier
- Website URL
- WordPress API connection status
- Last content publication date
- Creation and update timestamps

<Note>
  The connection status indicator helps you quickly identify which clients need attention for their WordPress integration.
</Note>

## Managing Clients

### Adding a New Client

1. Click the "Add New Client" button
2. Fill in the required information:
   - Client name
   - Website URL
   - WordPress API key
   - Koala Domain ID

<Warning>
  Make sure to enter the correct WordPress API key and Koala Domain ID to ensure proper integration functionality.
</Warning>

### Editing Client Details

To edit a client's information:

1. Find the client in the table
2. Click the edit icon
3. Update the necessary fields
4. Save your changes

### Removing a Client

<Warning>
  Removing a client will delete all associated data. This action cannot be undone.
</Warning>

To remove a client:

1. Locate the client in the table
2. Click the delete icon
3. Confirm the deletion

## Client Integration Status

The connection status indicates the health of your client's WordPress integration:

| Status | Meaning | Action Required |
|--------|---------|----------------|
| Connected | Active WordPress integration | None - working as expected |
| Disconnected | Integration needs attention | Check WordPress API key |
| Pending | Setup in progress | Complete integration setup |

## Best Practices

### Client Management

1. **Regular Audits**: Periodically review your client list to:
   - Verify connection status
   - Update outdated information
   - Clean up inactive clients

2. **Integration Monitoring**: Check connection status regularly to:
   - Identify disconnected clients
   - Resolve integration issues promptly
   - Ensure content delivery isn't interrupted

3. **Documentation**: Keep detailed records of:
   - Client onboarding dates
   - Integration updates
   - Connection issues and resolutions

<Tip>
  Sort the client table by "Last Published" to quickly identify clients who haven't received content recently.
</Tip>

## Understanding Client Data

Your client table displays several key pieces of information:

| Field | Description | Importance |
|-------|-------------|------------|
| Name | Client's business name | Primary identifier |
| Website | Client's website URL | Content destination |
| Status | WordPress connection state | Integration health |
| Last Published | Recent content delivery | Activity tracking |

<Info>
  All client data is automatically synced with your WordPress and Koala integrations.
</Info>

## Getting Support

If you encounter issues with client management:

- Check the connection status for integration problems
- Verify WordPress API credentials
- Contact support for persistent issues

<Warning>
  If multiple clients show as disconnected, there might be a system-wide integration issue. Contact support immediately.
</Warning>
