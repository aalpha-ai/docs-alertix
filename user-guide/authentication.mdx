# Authentication

DDS Blog AI uses Clerk, an enterprise-grade authentication system, to provide secure access while maintaining a seamless experience for dental practice content managers.

## Sign In

<Frame>

- **DDS Blog AI Logo**: Professional dental-themed logo

### Authentication Flow

<Steps>
  <Step title="Enter Email">
    Use the email address associated with your DDS Blog AI account.
  </Step>
  <Step title="Password Entry">
    Enter your secure password to access your account.
  </Step>
  <Step title="Multi-Factor Authentication">

  </Step>

<Tip>

</Tip>

## Security Features

DDS Blog AI prioritizes the security of your dental practice content management:

### Core Security Measures

- **Clerk Authentication**: Enterprise-grade security for all authentication processes
- **Role-Based Access**: Granular control over user permissions and content access
- **Email Verification**: Mandatory verification for all new accounts
- **Password Security**: Industry-standard encryption and secure storage
- **Session Management**: Automatic timeouts and active session monitoring

### Additional Protection

| Feature | Benefit | Recommended For |
|---------|----------|----------------|
| Multi-Factor Auth | Enhanced account security | Agencies & multi-client users |
| IP Allowlisting | Restricted access points | Enterprise clients |
| Audit Logging | Activity tracking | Compliance requirements |
| Auto-lockout | Brute force prevention | All accounts |

<Info>
  We recommend enabling all available security features, especially if you manage content for multiple dental practices.
</Info>

## Account Management

### User Roles

DDS Blog AI supports multiple user roles to match your organization's structure:

- **Admin**: Full platform access and client management
- **Content Manager**: Content creation and publishing rights
- **Client**: Limited access to specific practice content
- **Reviewer**: Content approval capabilities

<Note>
  User roles can be customized to match your dental practice or agency workflow.
</Note>

### Password Management

Maintain account security through proper password practices:

1. Regular password updates (recommended every 90 days)
2. Unique passwords for each user
3. No sharing of login credentials
4. Immediate deactivation of unused accounts

<Warning>
  Never share your login credentials, even with team members. Each user should have their own account with appropriate permissions.
</Warning>
